# ScoreFlux — instruções do projeto

SaaS de análise de crédito e fluxo de caixa para o agro. Replica em código a lógica das
planilhas de referência da raiz (`PADRAO_PLANILHA1.1.xlsx` e `Fluxo de Pagamentos - Estimativa.xlsx`).
Mapeamento completo das abas/fórmulas: **docs/01-mapeamento-planilhas.md** (leia antes de mexer em regra de negócio).

## Stack e estrutura

- `backend/` — Java 21, Spring Boot 3.3, Maven. Pacotes: `domain` (JPA), `repository`,
  `service` (regras), `api` (controllers + `api/dto`), `security` (JWT).
- `frontend/` — React 18 + TypeScript estrito + Vite. Uma página por item do menu lateral
  (`src/*Page.tsx`), montadas em `App.tsx`. HTTP centralizado em `src/api.ts`.
- Banco: H2 em arquivo (`backend/data/scoreflux.mv.db`) em modo PostgreSQL; perfil `postgres`
  pronto. Migrations Flyway em `backend/src/main/resources/db/migration/V{n}__nome.sql` —
  SQL portável H2/Postgres, identidade via `BIGINT GENERATED BY DEFAULT AS IDENTITY`.

## Regras de arquitetura (não negociáveis)

1. **Nenhum valor calculado é persistido.** Totais, saldos, score, indicadores = funções puras
   em `service/*Calculator.java`, cobertas por teste unitário. Exceção única: snapshot de
   análise CONCLUIDA (auditoria).
2. **BigDecimal** para dinheiro no backend; nunca double.
3. Multi-tenant futuro: toda tabela nova tem `empresa_id` (ou chega a ele via FK).
4. Política de crédito é **versionada** — nunca alterar pesos da v1; criar versão nova.
5. Nomes de domínio em português (cliente, analise, parecer), código idiomático Java/TS.
6. Controllers finos com records de request/response aninhados; erros via `ApiExceptionHandler`
   (`EntityNotFoundException`→404, `IllegalArgument/IllegalState`→400).

## Como rodar e testar

- Testes backend: `cd backend && mvn test` (integração usa H2 em memória + `TestAuth.autenticar(rest)` —
  novos testes de integração PRECISAM disso, tudo exige JWT).
- Build frontend: `cd frontend && npm run build` (tsc estrito — precisa passar limpo).
- Backend dev: NUNCA use `pkill -f spring-boot` (mata o próprio shell). Use:
  `lsof -ti :8080 | xargs -r kill` → depois
  `cd backend && nohup mvn -q org.springframework.boot:spring-boot-maven-plugin:run > /tmp/scoreflux-backend.log 2>&1 &`
- Frontend dev: gerenciado pelo painel de preview (porta 5173, proxy `/api`→8080).
- Login: `admin@scoreflux.com` / `admin123` (seed). API: `Authorization: Bearer <token>`,
  login em `POST /api/auth/login`.

## Convenções do frontend

- Formulários: componente `Campo` (label acima) e `InputMoeda` (máscara R$ padrão bancário,
  dígitos/100) de `src/ui.tsx`. Formatação: `brl`, `pct`, `num` de `api.ts`.
- Não introduzir libs novas sem necessidade (gráficos = SVG à mão; ver `NcgPage.tsx`).
- Paleta: verde `#1c6b33`, fundo `#f4f6f4`, erro `#a1352a` — não mudar cores.
- Cada página nova: tipo+métodos em `api.ts` → `XPage.tsx` → aba em `App.tsx` → estilos em `styles.css`.

## Glossário planilha → sistema

| Na planilha/transcrição | No sistema |
|---|---|
| "ROI" | ROE = Lucro Líquido / PL |
| "RCG" | CDG (Capital de Giro, modelo Fleuriet) |
| Aba NCG | `Demonstrativo` (contas granulares; agregados SEMPRE derivados) |
| Peso Atribuído (0–100 por opção) | nota da opção × peso do subcritério / 100 |
| Rating AAA–H | `ScoreCalculator.FAIXAS` (16 faixas) |
| Limite de crédito | média(PL, Receita Bruta últ. exercício) × % do rating |
